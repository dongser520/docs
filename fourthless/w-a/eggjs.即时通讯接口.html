                                  # 七、处理好友的申请（对申请加我为好友的信息进行处理）

                                  说明：（登录用户才能处理好友的申请，（游客）不能处理好友的申请）
                                  接口文档查看：五、处理好友的申请（对申请加我为好友的信息进行处理）

                                  1. 请求方式：post [用postman测试]或者[用Apipost测试]
                                  2. 接口示例：http://127.0.0.1:7001/api/chat/handleapply/:id
                                    :id表示申请表goodfriendapply表的id值
                                    本地路由地址：http://127.0.0.1:7001/api/chat/handleapply/1 (opens new window)
                                    1表示申请表goodfriendapply第1条申请信息
                                  3. header头传token

                                  请求参数 [Headers] -> [Key: token, Value: token值]

                                  参数 是否必填 类型 长度 说明
                                  token 是 string 由服务器生成 token令牌，如：eyJhbGciO.....
                                  1. 请求参数 [body] -> [x-www-form-urlencoded]
                                  参数 是否必填 类型 长度 说明
                                  id 是 int int(11) 申请表goodfriendapply表的id值,即：处理哪一条申请信息
                                  nickname 否 string 1-50位 好友备注，如：阿三
                                  status 是 枚举：ENUM['pending', 'refuse', 'agree', 'ignore'] 处理状态（结果）
                                  1. 返回示例
                                  {
    "msg": "ok",
    "data": "ok"
}
